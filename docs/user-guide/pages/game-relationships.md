## 左侧名册 (Roster)

→ 模板 `.rel-roster`

### 头部

| 元素 | 说明 |
|------|------|
| 标题 | "社交关系" |
| 计数徽章 | 总 NPC 数量 |
| [+] 按钮 | 打开添加 NPC 弹窗（空白 NPC 表单） |

### 搜索框

| 控件 | 类型 | 说明 |
|------|------|------|
| 搜索 NPC | 文本输入 | 按名称 / 类型 / 位置模糊匹配过滤 |

### 排序 Chip 栏

7 个排序维度，点击切换；再次点击同一维度翻转升/降序。当前选中 chip 高亮+箭头。排序状态持久化到 `localStorage`。

| Chip | 排序规则 |
|------|---------|
| 首字母 | `名称.localeCompare()` |
| 亲密度 | `好感度` 降序 |
| 性别 | `性别` 字母序，同性别内按名称 |
| 重点角色 | `是否主要角色` 优先，其次名称 |
| 最近互动 | `最后互动时间` 降序 |
| 位置 | `位置` 字母序 |
| 在场 | `是否在场` 优先 |

所有排序模式下，**已关注的 NPC 始终置顶**。

### NPC 卡片

```
┌─────────────────────────────────────────────────────┐
│  ┌──────┐  柳如烟                          ♡ 75    │
│  │ 头像  │  在场 · 同伴                    MAIN    │
│  │ (sm)  │  翠湖·酒楼                              │
│  └──────┘                                          │
└─────────────────────────────────────────────────────┘
```

| 元素 | 说明 |
|------|------|
| 头像 | `ImageDisplay` sm 尺寸。读 `图片档案.已选头像图片ID`，无则首字 fallback |
| 名称 | `名称` 字段 |
| 在场状态 | 读 `是否在场`：绿色"在场" / 灰色"离线" |
| 类型标签 | 读 `类型`。颜色分类：同伴=sage / 商人=amber / 敌对=red / 中立=灰 / 默认=灰 |
| 位置 | 读 `位置`，超过两级用 `·` 分隔时只取末尾两段 |
| 好感度 | `♡ 数值`，颜色：≤30 红 / ≤60 琥珀 / >60 绿 |
| MAIN 徽章 | 仅 `是否主要角色 === true` 时显示 |
| 关注高亮 | `关注 === true` 时卡片有关注样式 class |
| **点击** | 选中此 NPC → 右侧详情面板切换 |

### 类型统计

面板底部，按 NPC 类型汇总计数。如：`同伴 2  普通 1  敌对 1`。

### 空状态

无 NPC 时显示"暂无 NPC"。搜索无结果时显示"无匹配"。

| 按钮 | 功能 | 说明 |
|------|------|------|

## NPC 私聊 Modal

→ 组件 [NpcChatModal](../../../src/ui/components/shared/NpcChatModal.vue)
→ 管线 [NpcChatPipeline](../../../src/engine/pipeline/sub-pipelines/npc-chat.ts) `chat()` / `rollbackLastChat()`

### Header

| 元素 | 说明 |
|------|------|
| NPC 头像 | 首字符圆形 fallback |
| 名称 | "与 {名称} 私聊" |
| 元信息 Chip | 类型标签 + 📍位置 + ♡好感度（颜色：≤30 红 / ≤60 琥珀 / >60 绿） |
| [×] 关闭 | 关闭 Modal，不清空输入（切回后保留草稿） |

### 消息列表

| 元素 | 说明 |
|------|------|
| 空状态 | 聊天图标 + "还没有对话 — 输入点什么开始吧" |
| 玩家气泡 | 右对齐蓝色，`FormattedText` 渲染，底部显示时间戳 |
| NPC 气泡 | 左对齐灰色，`FormattedText` 渲染，底部显示时间戳 |
| 流式气泡 | 发送期间显示：有 chunk 时逐字渲染；无 chunk 时显示 3 点 loading 动画 |
| 时间戳格式 | 今天 → "HH:mm" / 昨天 → "昨天 HH:mm" / 同年 → "M月D日 HH:mm" / 更早 → "YYYY-M-D HH:mm" |
| 自动滚动 | 打开 Modal / 切换 NPC / 收到新消息 → 自动滚到底部 |
| NPC 消失检测 | 目标 NPC 从关系数组中消失时自动关闭 Modal + Toast 警告 |

| 元素 | 条件 |
|------|------|
| 错误文字 | `success: false` 时显示 AI 错误信息 |
| [×] 关闭 | 手动清除错误 |
| 滑入动画 | `Transition name="slide"` |

### 输入区

| 控件 | 类型 | 说明 |
|------|------|------|
| 文本域 | textarea 2 行 | placeholder: "对 {名称} 说点什么…（Enter 发送，Shift+Enter 换行）"。发送中 disabled |
| [撤回] 按钮 | button | 仅上一条对话可回退时显示。发送中 disabled。琥珀色调 |
| [发送] 按钮 | button | 输入为空或发送中 disabled |
| Enter 发送 | 键盘 | Enter = 发送，Shift+Enter = 换行，IME 输入法中不触发 |

### 撤回（回退）功能

> → `NpcChatPipeline.rollbackLastChat()`

**回退范围：** 最近一次完整聊天交互（用户消息 + AI 回复 + 好感度等命令变更 + NPC 记忆写入 + Engram 事件）

| 行为 | 说明 |
|------|------|
| 撤回成功 | 状态树恢复到发送前的快照 → 消息列表自动减少 → Toast "已撤回上一条对话" |
| 回合已推进 | 如果聊天后玩家推进了主回合，撤回被阻止 → Toast "快照期间已推进回合，回退可能导致数据丢失，已阻止" |
| 无可撤回 | 按钮不显示（首次发消息前或已撤回后） |
| 连续发 2 条 | 只能撤回最后 1 条（快照被覆盖） |
| 刷新后 | 快照不跨页面保留（存于内存，不进状态树） |

### Engram 集成

私聊完成后，NPC 的回复文本和命令变更会通过 `engramManager.processResponse()` 写入知识图谱。撤回时通过 `syncVectorsToState()` 清理孤立向量。此过程对用户不可见，仅影响 Engram 的事件和实体数据。
