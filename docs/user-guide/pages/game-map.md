## 地图视口 (Cytoscape)

→ 模板 `.cy-viewport`
→ Cytoscape.js `fcose`（Fast Compound Spring Embedder）布局，专为 compound node 设计，原生避免兄弟节点/容器重叠

### 节点类型

| 类型 | 视觉 | 条件 |
|------|------|------|
| 容器节点 (parent) | 圆角矩形，半透明背景，标签在顶部居中 | 有子地点的节点（`上级` 字段指向它） |
| 叶子节点 (childless) | 椭圆形 24×24，标签在底部 | 无子地点的末端节点 |

### 颜色系统

20 色族：根节点按名称 hash 分配色族，所有子孙继承同色族。同色 = 同区域树。

### 探索三态

| 状态 | 视觉 | 判定 |
|------|------|------|
| 已探索 | 绿色边框（opacity 0.7） | 地点名在 `世界.探索记录` 中 |
| 部分探索 | 黄色虚线边框（opacity 0.5） | 自身未探索但有已探索的后代 |
| 未探索 | 整体半透明（opacity 0.45） | 自身和所有后代均未探索 |

### 玩家位置

| 视觉 | 说明 |
|------|------|
| 红色呼吸外环 | 节点 outline：宽度 1-6px + 透明度 0.3-0.95 循环呼吸，周期 3s |
| 祖先容器红边框 | 玩家所在位置的所有祖先容器节点有红色边框（opacity 0.5） |
| 位置匹配 | 精确匹配 → `·` 前缀匹配（玩家可能在子地点，而地图只有祖先节点） |

### 节点标签

| 标签内容 | 说明 |
|------|------|
| 短名 | 取 `·` 分隔的最后一段（如 `青云城·翠湖·酒楼` → `酒楼`） |
| NPC 后缀 | 有 NPC 时追加 `(柳如烟、赵灵儿)` |

### 边（不可见）

兄弟链边（sibling-chain）：同父的子节点之间依序串联。`width: 0` + `opacity: 0` 完全隐藏。目的是让布局引擎把兄弟拉近，收紧容器 bounding box。

## 地图交互

| 操作 | 行为 |
|------|------|
| 滚轮 / 双指缩放 | 缩放地图。范围 0.15x - 6x |
| 拖拽空白处 | 平移画布 |
| 单击节点 | 选中 → 底部展开「地点详情面板」 |
| 单击空白 | 取消选中 → 关闭详情面板 |
| 双击容器节点 | Drill：聚焦到该区域+子孙，push 到 focus stack。带 350ms 动画 |
| 双击空白 | Pop：退回上一层。stack 清空时 fit 全图。带 300ms 动画 |
| 拖动节点/容器 | 自由移动节点或整个容器（含子孙） |
| 拖动松手（有重叠时） | 自动检测与兄弟节点的 bounding box 重叠，触发局部 fcose 重布局弹开重叠节点（200ms 动画） |
| 鼠标悬停节点 | 显示浮动 Tooltip |
| 鼠标移出节点 | 隐藏 Tooltip |

## 地点详情面板

→ 模板 `.location-detail`
→ 选中节点后在地图下方 slide 出现（`Transition name="detail-slide"`）

```
┌─ 翠湖·酒楼 ──────────────────────────────── [✕] ───────┐
│  [已探索]  [📍 你在这里]                                 │
│                                                          │
│  繁华的酒楼，常有江湖人士出入                             │
│  ── 或 ── 暂无描述                                       │
│                                                          │
│  类型    商业区              ← 有 类型 时显示             │
│  上级    翠湖                ← 有 上级 时显示，可点击导航 │
│  NPC     [柳如烟] [赵灵儿]   ← 有 NPC 时显示             │
│  内部    [大厅] [后院]       ← 有子地点时显示，可点击导航 │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

| 元素 | 说明 |
|------|------|
| 地点名 | `名称` 全名 |
| [✕] | 关闭详情面板 |
| 探索徽章 | "已探索"=绿 / "未探索"=灰 |
| 你在这里徽章 | 仅当玩家位置 === 地点名时显示，红色 |
| 描述 | `描述` 字段。无描述时显示斜体"暂无描述" |
| 类型 | `类型` 字段，有则显示 |
| 上级 | `上级` 字段。**可点击** → 选中上级地点并 animate fit |
| NPC | `NPC[]` 数组，tag 列表 |
| 内部 | 子地点列表（`上级 === 当前地点名称` 的地点）。每个子地点是**可点击按钮** → 选中并 animate fit |

面板最大高度 240px，内容超出时可滚动。

## 数据刷新机制

| 监听 | 行为 |
|------|------|
| 地点列表变化 | 监听 `validLocations` JSON 序列化字符串变化 → `refreshGraph()` 全量重建 |
| 玩家位置 / 探索记录变化 | 监听序列化拼接字符串变化 → `updateNodeStyles()` 仅更新节点样式（不重新布局） |
| `onActivated`（KeepAlive 恢复） | 强制 `refreshGraph()` 重建，保证切回地图 Tab 数据始终最新 |
