# Prompt 组装调试面板

> → [src/ui/components/panels/PromptAssemblyPanel.vue](../../../src/ui/components/panels/PromptAssemblyPanel.vue)
>
> 游戏内调试面板。展示引擎向 AI 发送的完整 prompt 快照，包含所有消息、
> 模板变量、来源标签、CoT 推理内容和原始 AI 输出。

## 1. 顶栏

| 元素 | 说明 |
|------|------|
| 标题 | "Prompt 组装调试" |
| [导出快照] | 将当前选中的快照导出为 JSON 文件。无数据时灰化 |
| [清除全部] | 打开清除确认弹窗。无数据时灰化 |

---

## 2. 快照缓冲

| 元素 | 说明 |
|------|------|
| 容量 | 仅保留最近 1 次快照，每次新调用自动覆盖 |
| 无 Tab 切换 | 因为只有 1 个快照，Tab 栏不显示 |

---

## 3. 摘要栏（Summary Bar）

水平排列的 4 个指标。

| 指标 | 格式 | 说明 |
|------|------|------|
| Flow | 等宽字体 | 当前快照的 flowId，如 `mainRound` / `openingScene` / `npcChat` / `image:imageCharacterTokenizer` |
| 消息数 | 等宽字体 | 快照中的 message 数量 |
| 预估 Tokens | 等宽字体 | `~4,231` 格式。CJK 字符按 1.5 字/token 计算，其他按 4 字符/token |
| CoT | 仅有推理时显示 | 紫色 "🧠 有推理" 标记 |

---

## 4. 快照元数据

```
  Round 5        2026-05-04 14:32:01        gen: abc123
```

| chip | 说明 |
|------|------|
| Round N | 当前游戏回合号（仅有值时显示） |
| 时间戳 | 快照捕获时的完整日期时间 |
| gen: ID | generationId，等宽字体，用于关联引擎日志 |

---

## 5. 来源分解（Source Breakdown）

一行彩色 chip 展示该请求由哪些 prompt 部件组成，无需逐条展开即可一览。

```
  组成部件（共 18 条消息）
  [AI角色声明 ×1] [世界观 ×1] [在场角色 ×1] [叙事约束+风格 ×1]
  [长期记忆 ×1] [Narrative·User ×3] [Current Input ×1] ...
```

每个 chip 显示：

| 元素 | 说明 |
|------|------|
| 标签 | 部件中文名，等宽字体 |
| 数量 | `×N`，半透明 |
| 颜色 | 按来源类别区分（见下表） |
| tooltip | 悬停显示详细说明 |

### 5.1 来源类别颜色表

|-------------|------|------|------|

| `builder:*` | 紫罗兰 #8b5cf6 | SystemPromptBuilder 的上下文片段 | `builder:ai_role`, `builder:memory_long` |
| `short_term_memory` | 青色 #06b6d4 | 即时剧情回顾（短期记忆） | — |
| `history:user` | 靛色 #6366f1 | 叙事历史中的玩家输入 | — |
| `history:assistant` | 绿色 #22c55e | 叙事历史中的 AI 回复 | — |
| `current_input` | 粉色 #ec4899 | 本回合玩家输入 | — |
| `placeholder` | 石板灰 #64748b | Gemini API 占位消息 | — |
| `step1_thinking_context` | 紫色 #a855f7 | Step1 CoT 注入 step2 | 仅分步生成 |
| `step1_response` | 青绿 #14b8a6 | Step1 输出注入 step2 | 仅分步生成 |
| `step2_followup` | 橙色 #f97316 | Step2 指令 | 仅分步生成 |

## 7. 原始 AI 输出区

仅当快照包含 rawResponse 时显示。默认折叠。

## 10. 消息列表

每条消息是一个可折叠的卡片。

### 10.1 消息头部（可点击折叠）

```
  SYSTEM  #1  [AI角色声明]  · 长期记忆 · 中期记忆 · Engram·事件    ~1200 tokens
```

| 元素 | 颜色 | 说明 |
|------|------|------|
| 角色 badge | SYSTEM=琥珀 / USER=紫色 / ASSISTANT=绿色 | 大写字母 |
| 序号 | 灰色等宽 | `#1`, `#2`, ... |
| 来源 badge | 与来源分解同色 | 该消息的 source tag 中文名，悬停 tooltip |
| 记忆子段 chip | 仅 system + 含记忆时 | 显示各记忆层名称（如"长期记忆"、"Engram·事件"） |
| token 估算 | 灰色等宽 | `~1200 tokens`，右对齐 |
| 折叠箭头 | 灰色 | 旋转动画 |

### 10.2 消息内容

展开后显示：

1. **记忆段导航**（仅检测到记忆子段时）：

```
  ● 长期记忆      行 42 – 88
  ● 中期记忆      行 90 – 134
  ● 隐式中期      行 136 – 148
  ● Engram·事件   行 150 – 162
  ● Engram·实体   行 164 – 180
```

每行左侧彩色圆点 + 名称 + 行号范围（等宽字体）。

2. **完整文本**：等宽字体，深色背景，最大高度 400px 可滚动。

### 10.3 记忆子段识别规则

通过 markdown header 模式匹配，识别系统消息中的记忆段落：

| markdown header | 子段类型 | 颜色 |
|----------------|---------|------|
| `### 长期记忆…` | long_term | 紫 #8b5cf6 |
| `### 中期记忆…` | mid_term | 蓝 #0ea5e9 |
| `### 隐式记忆…` | implicit_mid_term | 青 #06b6d4 |
| `### 短期记忆…` | short_term | 绿 #22c55e |
| `#### 相关事件记忆` | engram_events | 橙 #f97316 |
| `#### 相关角色/实体` | engram_entities | 洋红 #d946ef |
| `#### 关系网络` | engram_relations | 粉 #ec4899 |
| `#### 相关规则` | engram_rules | 黄 #eab308 |
