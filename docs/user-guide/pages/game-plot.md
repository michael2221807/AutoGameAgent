# 剧情导向面板 (Plot Panel)

> → [src/ui/components/panels/PlotPanel.vue](../../../src/ui/components/panels/PlotPanel.vue)
>
> 剧情弧线的完整管理界面。路由 `/game?panel=plot`。
> 用户在这里创建剧情大纲，管理节点链、度量值，查看 AI 评估日志，
> 以及在关键节点出现时做出推进/拒绝决定。

## 弧线头部 (Arc Header)

> → `PlotPanel.vue` section `.arc-header`

弧线头部使用 glassmorphism 卡片样式（`--glass-bg` + `--glass-blur` + `--glass-shadow`）。

### 弧线信息

| 元素 | 说明 |
|------|------|
| 弧线标题 | 衬线字体 (`--font-serif-cjk`)，加粗 |
| 状态徽章 | `draft` / `active` / `completed` / `abandoned`，不同背景颜色 |
| 概要描述 | 如有 synopsis 则显示 |
| ✏ 编辑图标 | 弧线标题/概要旁显示编辑图标（Story 2）。点击进入内联编辑模式，blur/Enter 提交，Escape 取消。通过 `usePlotEditor.updateArc()` composable 保存 |

### 状态徽章颜色

| 状态 | 背景 | 文字色 |
|------|------|--------|
| draft | 白色半透明 | 次要文字色 |
| active | sage 绿半透明 | sage-400 |
| completed | amber 半透明 | amber-400 |
| abandoned | 危险红半透明 | danger |

### 弧线操作按钮

| 按钮 | 条件 | 说明 |
|------|------|------|
| 激活弧线 | draft 且有节点 | 将弧线从草稿变为活跃 |
| 重新激活 | abandoned 且有节点 | 将已放弃弧线重新激活 |
| 放弃弧线 | active | 打开 `确认放弃弹窗` |
| 删除 | 非 active | 直接删除弧线 |

### AI 拆解状态

| 状态 | 说明 |
|------|------|
| 拆解中 | 绿色文字 "AI 正在拆解大纲..." |
| 拆解失败 | 红色文字，显示错误信息 |


### 节点条目

每个节点显示为时间线样式：左侧状态图标 + 连线，右侧内容体。

| 元素 | 说明 |
|------|------|
| 状态图标 | `completed`(绿色对勾) / `active`(蓝色圆点) / `skipped`(跳过符号) / `pending`(空心圆) |
| 节点标题 | 点击打开 `节点详情弹窗` |
| 回合信息 | 仅 active 节点：`[已用轮/最大轮]` |
| 关键徽章 | importance = `critical` 时显示红色 "关键" 标记 |
| 叙事目标 | 仅 active 节点显示 `narrativeGoal` |
| [在此之后插入] | 在该节点后方插入新节点 |
| [删除] | 移除该节点 |

### 节点交互

| 操作 | 说明 |
|------|------|
| 点击节点体 | 触发 `select` 事件，打开节点详情弹窗 |
| 点击"在此之后插入" | 触发 `insert-after(index)` 事件，打开添加节点弹窗并设定插入位置 |
| 点击"删除" | 触发 `remove(nodeId)` 事件，直接删除 |

## 数据流

| 来源 | 路径 | 说明 |
|------|------|------|
| 剧情状态 | `DEFAULT_ENGINE_PATHS.plotDirection` | 完整 `PlotDirectionState` |
| 当前回合 | `DEFAULT_ENGINE_PATHS.roundNumber` | 数字 |
| 评估日志 | `plotDirection._evalLog` | `PlotEvalLog[]` |
| 评估中标志 | `plotDirection._evaluating` | boolean |
| 评估日志开关 | `系统.设置.plot.showEvalLog` | boolean |

**持久化机制：** 每次修改后调用 `persist()` → 将 store 快照写回状态树 → 发送 `engine:request-save` 事件触发 IndexedDB 写入。
