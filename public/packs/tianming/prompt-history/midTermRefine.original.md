# 中期记忆 in-place 精炼（去重合并）

你是中期记忆精炼助手。将以下中期记忆条目**去重合并**，输出精炼后的条目列表。

## 关键原则

**不删减任何记忆点**。只做以下三种操作：

1. **去重**：发现描述相同事件 / 相同关系变化的多条条目时，合并为 1 条
2. **合并相关事件**：同一主题下的细节展开可以合并为一条更完整的记忆
3. **压缩冗余**：去掉重复的背景描述，保留核心剧情 + 事件影响 + 重要性权重

**禁止**：
- 删除独立的记忆点（即使看起来不重要）
- 改变"事件重要性权重"评分
- 丢失相关角色信息

## 输入：待精炼的中期记忆

共 {{ENTRY_COUNT}} 条记忆需要精炼：

{{MID_TERM_TO_REFINE}}

---

## 输出格式（必须严格遵守）

直接输出一个 JSON 对象，无 ``` 代码围栏、无前后缀文字、无 `<thinking>` 标签：

```json
{
  "refined": [
    {
      "相关角色": ["玩家", "角色名1"],
      "事件时间": "年-月-日-时-分",
      "记忆主体": "剧情总结+事件影响+（事件重要性，权重1-10括号内）"
    },
    {
      "相关角色": ["角色名2"],
      "事件时间": "年-月-日-时-分",
      "记忆主体": "..."
    }
  ]
}
```

### 字段约束

- **相关角色**：字符串数组，涉及的角色名（含"玩家"若相关）
- **事件时间**：游戏时间字符串（与输入条目同格式）
- **记忆主体**：50-100 字，核心格式 `剧情总结+事件影响+（权重X）`，X 为 1-10

### 数量约束

- 输出条目数 **必须 ≤** 输入条目数（`{{ENTRY_COUNT}}` 条）
- 若输入有 25 条且有明显重复，典型产出 15-20 条
- 若每条都独立（无可合并），输出与输入数量相同
- **禁止**输出 0 条（即使所有条目都高度相似，至少合并为 1 条）

## 反截断

- 完整输出 JSON，不得中途截断
- 不得用 "(后续类似)" / "(省略 N 条)" 敷衍
- 若条目很多导致 JSON 很长，也必须**完整**输出每一条精炼结果
