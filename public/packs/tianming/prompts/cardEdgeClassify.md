## 分类规则

- **worldview（世界观设定）**：不依赖具体剧情发生也成立的稳定事实——身份、归属、性格、长期关系、地点属性、组织结构。例如「张三是天剑门嫡传弟子」「东城茶馆位于城东集市旁」。
- **plot-event（剧情事件）**：由游玩过程产生、带有时间性的事件结果——冲突、约定、状态变化、一次性事件。例如「张三上周与玩家发生争执」「玩家答应三日后赴约」。

判断要点：

1. 人物↔人物、地点↔人物的稳定关系是世界卡的核心，优先判为 worldview。
2. 边上标注的来源与回合信息仅作参考，以事实文本本身的语义为准。
3. 拿不准时判为 plot-event（宁缺勿滥，玩家会逐条复核）。

## 输出格式（最高优先级）

**只输出一个合法 JSON 对象：**

```json
{"edge_classifications": [
  {"edge_id": "张三|天剑门|张三是天剑门的嫡传弟子师从", "category": "worldview"},
  {"edge_id": "玩家|张三|玩家上周与张三发生争执不欢而散", "category": "plot-event"}
]}
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `edge_id` | ✅ | 直接复制上方 `[...]` 中的边 ID，不要自行构造 |
| `category` | ✅ | `worldview` 或 `plot-event` |

### 严禁

- ❌ `<thinking>` / `<reasoning>` 标签
- ❌ JSON 对象之外的任何文字
- ❌ 编造不存在的 edge_id（只分类上方列出的边）
- ❌ 遗漏任何一条边（每条边都必须出现在输出中）
