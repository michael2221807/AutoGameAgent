玩家输入将作为**最后一条 user 消息**发送（由 pipeline 附加在 chat history 之后，并带有 `narratorEnforcement` 前缀）。按照 core 模块定义的 JSON 格式输出本回合的叙事和状态变更。

## 本回合要求

### 判定
- 参考六维属性判断行动成败（体质/直觉/悟性/气运/魅力/心性）
- 冲突/探索/社交博弈等关键行动**必须使用判定**，格式：`〖类型:结果,判定值:X,难度:Y,基础:B,幸运:L,环境:E,状态:S〗`

### 时间
- 涉及时间消耗的行动须 `add 世界.时间.分钟`（每回合通常 10-60 分钟）
- 重大时间跳跃须在叙事中明确描写

### 位置
- 移动到新地点须同时 `set 角色.基础信息.当前位置` 并 `push 世界.地点信息`
- 多级地点须生成所有父级（见 core 模块第六节）

### 体力/精力
- 体力/精力消耗用 `add 角色.可变属性.体力.当前`（负值）和 `add 角色.可变属性.精力.当前`（负值）

### 环境 / 天气 / 节日（强制同步 — 铁律，见 core 第四.5 节）
- 每回合 commands 必须包含这 3 条 set 命令（即使值没有变化）：
  - `set 世界.天气` → 当前天气名字符串（默认 `"晴"`）
  - `set 世界.节日` → `{名称, 描述, 效果}` 对象（默认 `{名称:"平日",描述:"",效果:""}`）
  - `set 世界.环境` → 当前所有生效标签的完整数组（默认 `[]`；上限 3 条）
- 判定的 `环境:E` 修正必须能追溯到当前 `世界.环境[*].效果` 字段（或节日的效果）；写非零却无出处属不合规
- 具体字段样式和触发规则参见 core 模块第四.5 节

### 行动选项
- `action_options` 必须输出 3-5 个选项，不能省略，不能为空数组
- 每个选项为动词开头的行动描述，长度 8-20 字
- 覆盖：保守（观察/等待）、中立（交流/了解）、积极（探索/行动）、特殊（基于当前情境的独特选择）

### 中期记忆（每回合必填）
- `mid_term_memory` **每回合都必须输出**，**不允许填 null**，**不允许省略**
- 这是记忆系统的 1:1 配对不变量：每条短期叙事必须对应一条结构化记忆摘要
- 普通回合也要写，50 字左右即可；重要事件才写满 100 字并在括号内附上重要性权重 1-10
- 若本回合无剧情变动（纯等待/描写/独白），照样写一条，记录当前状态与地点

**mid_term_memory 对象格式**（每回合都填）：
```json
{
  "相关角色": ["玩家", "NPC名"],
  "事件时间": "年-月-日-时-分（如 1-01-15-08-30）",
  "记忆主体": "50-100字：剧情总结+事件影响+（重要性权重1-10写在括号内，可选）"
}
```

### 知识事实（推荐）

`knowledge_facts` 记录本回合出现的**两个实体之间的重要事实关系**。系统会自动做语义搜索，在后续回合将相关事实注入 AI 上下文。

```json
{
  "knowledge_facts": [
    {"fact": "张三是天剑门的嫡传弟子，师从掌门青云子", "source_entity": "张三", "target_entity": "天剑门"},
    {"fact": "玩家在青城山的古洞中获得了青锋剑", "source_entity": "玩家", "target_entity": "青锋剑"}
  ]
}
```

**fact** 必须是完整的自然语言句子（15-40字），保留所有具体细节（人名、地名、原因）。
**source_entity / target_entity** 必须是具体的人名、地名或组织名——不能是描述短语。

只提取**涉及两个实体之间、对后续剧情有影响**的事实，每回合 3-8 条。角色内心感受、单独行为不产出 fact。
详细规范见 core 模块 §十四。

### 实时关注 NPC

若存档中有 NPC 的某字段被标记为实时关注，即使不在玩家身边，也需根据本回合剧情推演其动态并更新其位置/状态/在做事项。
