### 1. NPC `私密信息` 必填字段

每个 NPC 的 `私密信息` 对象**必须**包含以下全部字段，不得缺失或使用「待生成/暂无/无」等占位：

```json
"私密信息": {
  "是否为处女/处男": true,
  "身体部位": [
    {"部位名称": "嘴",   "敏感度": 40, "开发度": 10, "特征描述": "...", "特殊印记": ""},
    {"部位名称": "胸部", "敏感度": 60, "开发度": 20, "特征描述": "...", "特殊印记": ""},
    {"部位名称": "小穴", "敏感度": 50, "开发度": 0,  "特征描述": "...", "特殊印记": ""},
    {"部位名称": "屁穴", "敏感度": 30, "开发度": 0,  "特征描述": "...", "特殊印记": ""}
  ],
  "性格倾向": "...",
  "性取向": "...",
  "性癖好": ["..."],
  "性渴望程度": 30,
  "性交总次数": 0,
  "性伴侣名单": []
}
```

**身体部位（硬约束）**：数组必须至少包含 4 个固定部位条目，`部位名称` 必须精确为 `嘴`、`胸部`、`小穴`、`屁穴`。可以在这 4 项之后根据 NPC 特征自由追加更多条目（如 乳首 / 臀部 / 后穴 / 阳具 / 脚趾 等），但 4 个固定项不得缺失。每条的 `特征描述` 必须写出具体形态/触感（50-120 字），生图流程会据此生成特写图。

**非处女/处男时追加必填 3 字段**：当 `是否为处女/处男 === false`，`私密信息` 还必须包含：

```json
{
  "初夜夺取者": "具体人名 / '童年被拐' / '师门内乱' 等情境描述",
  "初夜时间": "游戏内时间或自然描述，如 '14 岁冬'、'入赘前夜'",
  "初夜描述": "50-200 字：地点、氛围、关系性质、NPC 当时感受"
}
```

`是否为处女/处男=true` 时这 3 字段可省略。

### 2. 逻辑一致性（硬约束）

- `是否为处女/处男=true` → `性交总次数=0`、`性伴侣名单=[]`；**`初夜夺取者/初夜时间/初夜描述` 必须省略或为空**
- `是否为处女/处男=false` → `性交总次数>=1`、`性伴侣名单` 不得为空（使用具体人名或"已故/失踪"）、**`初夜夺取者/初夜时间/初夜描述` 必须全部填写**
- 每个"身体部位"条目的 `敏感度` 和 `开发度` 须为 0-100 的数字
- `初夜夺取者` 应出现在 `性伴侣名单` 中（除非 `未知`/早年不可考情境）

### 3. 合理性

- 应根据 NPC 的背景/类型/年龄推导出符合设定的字段值
- 禁止给所有 NPC 填同一份模板数据
- 禁止使用"待生成"、"暂无"、"无"、"未知"等占位字符串
