# 增强开局 — 结构化响应（第2步）

基于开场叙事的内容，生成结构化响应。

## 必须输出的字段

### 1. commands
仅做剧情产生的微调：
- 可以 `set` 已有 NPC 的字段（位置、在做事项、是否在场等）
- 可以 `set` 玩家的当前位置
- **不要用 push 创建新 NPC**（已在前置阶段完成）
- **不要用 push 创建新地点**（已在前置阶段完成）

每条 command 的 JSON 结构：
```json
{"action": "set", "path": "社交.关系[名称=李明].位置", "value": "青云城·客栈"}
{"action": "set", "path": "社交.关系[名称=李明].在做事项", "value": "正在喝茶"}
{"action": "set", "path": "社交.关系[名称=李明].是否在场", "value": true}
{"action": "set", "path": "角色.基础信息.当前位置", "value": "青云城·客栈"}
```

### 2. action_options
必须 3-5 个首轮行动选项。玩家需要选择下一步行动。

### 3. mid_term_memory
必须输出一条开局记忆：
```json
{ "相关角色": ["玩家", ...相关NPC], "事件时间": "...", "记忆主体": "开局场景概述..." }
```

## 不要输出的字段

### 4. knowledge_facts
**不要输出此字段。** 结构性关系已在前置阶段生成。
如果你仍然输出了此字段，它将被丢弃。

## 输出格式（严格 JSON）

```json
{
  "commands": [
    {"action": "set", "path": "角色.基础信息.当前位置", "value": "地点名称"},
    {"action": "set", "path": "社交.关系[名称=X].是否在场", "value": true}
  ],
  "action_options": [
    "行动选项1（简短自然语言）",
    "行动选项2",
    "行动选项3"
  ],
  "mid_term_memory": {
    "相关角色": ["玩家", "NPC名1"],
    "事件时间": "",
    "记忆主体": "开局场景概述（50-100字）"
  }
}
```

- 禁止输出 text 字段（叙事已在第1步完成）
- 禁止 `<thinking>` 标签
- 只输出 JSON，不要输出任何解释文字
