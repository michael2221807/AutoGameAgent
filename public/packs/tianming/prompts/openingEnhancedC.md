## 关键约束

1. 每个 NPC 是一个完整档案对象，字段名必须使用下方输出格式中的**中文键名**。
   内容要求：
   - 名称：必填，唯一标识
   - 性别：男/女/其他
   - 年龄：数字
   - 类型："重点" 或 "普通"，建议主要角色为 "重点"
   - 好感度：-100 到 100 的整数
   - 位置：必须引用已有地点名称，用 · 分隔层级如 "青云镇·茶馆"
   - 描述：一句话身份/气质概述，30字以内
   - 外貌描述：完整外貌段（脸、发、瞳、体型、气质），50-200字
   - 身材描写：身高/体态/身体特征，30-120字
   - 衣着风格：日常着装偏好（材质、色调、款式），30-100字
   - 背景：出身/经历，50-200字
   - 内心想法：当前内心状态
   - 在做事项：当前正在做的事
   - 性格特征：2-4 个标签数组，如 ["温柔", "腹黑"]
   - 核心性格特征：一句话锚定主性格
   - 是否主要角色：布尔值 true/false
   - 关系状态：玩家与此 NPC 的关系，如"陌生人"/"朋友"/"家人"
   - 好感度突破条件：下一阶段好感提升条件
   - 关系突破条件：关系升级触发条件
   - 记忆：空数组 []
   - 关系网变量：空数组 []
{{NSFW_SECTION}}

2. **严禁输出 NPC 间关系**——关系网变量保持空数组 []。
   NPC 间关系由后续的知识边生成阶段统一处理。

3. 关系状态字段保留供 UI 展示，但不作为权威关系数据源。

4. 生成数量：{{NPC_MIN}}-{{NPC_MAX}} 个

## 输出格式（严格 JSON，禁止任何非 JSON 内容）

键名必须使用下方示例中的**中文名称**，严禁使用英文键名。

```json
{
  "npcs": [
    {
      "名称": "NPC中文名",
      "性别": "男",
      "年龄": 25,
      "类型": "重点",
      "好感度": 0,
      "位置": "青云镇·茶馆",
      "描述": "一句话身份/气质概述",
      "外貌描述": "完整外貌段（脸、发、瞳、体型等）",
      "身材描写": "身高/体态描写",
      "衣着风格": "日常着装偏好",
      "背景": "出身/经历",
      "内心想法": "当前内心状态",
      "在做事项": "当前正在做的事",
      "性格特征": ["标签1", "标签2"],
      "核心性格特征": "一句话锚定主性格",
      "是否主要角色": true,
      "关系状态": "陌生人",
      "好感度突破条件": "下一阶段好感提升条件",
      "关系突破条件": "关系升级触发条件",
      "记忆": [],
      "关系网变量": []
    }
  ]
}
```

- 禁止 `<thinking>` 标签
- 只输出 JSON，不要输出任何解释文字
