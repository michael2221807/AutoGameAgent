## 判断规则

对每条已有知识边，做出以下判断之一：

- **keep**：事实仍然成立，不做改动
- **invalidate**：事实已不成立（例如：关系已断裂、身份已变、物品已易手），标记失效

判断依据：结合上方 GAME_STATE_JSON 中的当前状态、记忆摘要、以及最近叙事中发生的事件。

---

## 输出格式（最高优先级）

**只输出一个合法 JSON 对象：**

```json
{"edge_updates": [
  {"edge_id": "张三|天剑门|张三是天剑门的嫡传弟子师从", "action": "invalidate", "reason": "张三已被逐出天剑门"},
  {"edge_id": "玩家|青锋剑|玩家在青城山获得了青锋剑", "action": "keep", "reason": "玩家仍持有青锋剑"}
]}
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `edge_id` | ✅ | 知识边 ID — 直接复制上方列出的边的 `[...]` 中的 ID，不要自行构造 |
| `action` | ✅ | `keep` 或 `invalidate` |
| `reason` | ✅ | 一句话说明判断理由 |

### 严禁

- ❌ `text` / `commands` / `mid_term_memory` / `action_options` 字段
- ❌ `<thinking>` / `<reasoning>` 标签
- ❌ JSON 对象之外的任何文字
- ❌ 编造不存在的 edge_id（只审查上方列出的边）
