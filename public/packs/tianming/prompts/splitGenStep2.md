# 分步生成 2/2：仅指令

## 输出格式（必须严格遵守）

只输出一个 JSON 对象，禁止任何前后缀文字、禁止 ` ``` ` 代码块：

```
{"mid_term_memory":{"相关角色":["张三"],"事件时间":"1-01-15-08-30","记忆主体":"50-100字"},"commands":[{"action":"add","path":"世界.时间.分钟","value":30}],"action_options":["选项1","选项2","选项3","选项4","选项5"],"knowledge_facts":[{"fact":"...","source_entity":"...","target_entity":"..."}]}
```

---

## mid_term_memory 结构（必填，对象格式）

- **相关角色**：本回合涉及的角色名数组（含玩家可写"玩家"），如 `["玩家","张三"]`
- **事件时间**：与游戏时间同形可排序串，如 `"1-01-15-08-30"`（年-月-日-时-分）
- **记忆主体**：50-100 字，格式「剧情总结 + 事件影响 +（重要性权重写在括号内）」

---

## JSON 与 path 规则（重要）

- 只输出一个 JSON 对象，禁止任何前后缀文字
- 字符串如需换行，用 `\n`
- 规则文中的 `[NPC名]` 只是占位符，输出 path 时**必须替换成真实名称**，且不要保留方括号
- 方括号 `[]` 只有数组索引表达式可以用：例如 `角色.效果[0]`、`社交.关系[名称=李明阳]`

---

## 本步骤需要输出

- `mid_term_memory`：必填，对象格式（如上）
- `commands`：数据更新指令数组
- `action_options`：3-5 个行动选项（字符串数组）
- `knowledge_facts`（推荐）：本回合重要事实

---

## NPC 操作规范

**创建新 NPC**（必须一次性 push 完整对象，带 `类型` 字段）：
```json
{"action": "push", "path": "社交.关系", "value": {
  "名称": "李明阳",
  "性别": "男",
  "年龄": 35,
  "出生日期": {"年": -10, "月": 3, "日": 12},
  "类型": "重点",
  "好感度": 50,
  "位置": "青云镇·茶馆",
  "描述": "外貌简述",
  "性格特征": ["沉稳", "多疑"],
  "关系状态": "陌生人",
  "记忆": [],
  "在做事项": "在茶馆里打探消息"
}}
```

[/私密]

**更新 NPC 字段**：
```json
{"action": "add", "path": "社交.关系[名称=李明阳].好感度", "value": 10}
{"action": "set", "path": "社交.关系[名称=李明阳].在做事项", "value": "赶往城门方向"}
{"action": "set", "path": "社交.关系[名称=李明阳].位置", "value": "华京市·德威贵族高中·校门口"}
{"action": "set", "path": "社交.关系[名称=李明阳].是否在场", "value": true}
{"action": "push", "path": "社交.关系[名称=李明阳].记忆", "value": "与玩家在茶馆相识，对其颇感兴趣"}
```

**NPC 位置同步（硬约束）**：
- 每回合叙事中出现的 NPC，**必须** `set` 其 `位置` 为当前所在地点（使用完整 `·` 分隔路径）
- NPC 跟随玩家移动时，`位置` 必须更新为玩家的新位置
- NPC 离开场景时，`位置` 更新为其去向，同时 `set 是否在场 = false`
- 这与 `角色.基础信息.当前位置` 同等重要——**位置缺失会导致 UI 无法显示 NPC 所在地**

**NPC-NPC 关系**（仅经 commands）：
```json
{"action": "set", "path": "社交.关系[名称=张三].关系.李四", "value": "师徒"}
```

## 地点与探索（必须）

- `角色.基础信息.当前位置` 变化时，必须同时 `push 世界.地点信息`
- 多级地点须依次生成全部父级（见 core 第六节）
- 地点仅限真实场所，禁止载具/状态/泛指（见 core 第六节）
- push 地点时须带 `描述`

---

## knowledge_facts 结构（推荐）

```json
{
  "knowledge_facts": [
    {"fact": "张三是天剑门的嫡传弟子，师从掌门青云子", "source_entity": "张三", "target_entity": "天剑门"},
    {"fact": "玩家在青城山获得了一柄青锋剑", "source_entity": "玩家", "target_entity": "青锋剑"},
    {"fact": "李四对玩家怀有深深的仇恨，因其父被玩家所杀", "source_entity": "李四", "target_entity": "玩家"}
  ]
}
```

- `fact`：完整自然语言句子（15-40字），保留所有具体细节
- `source_entity / target_entity`：必须是人名、地名或组织名
- 只提取**涉及两个实体之间、对后续剧情有影响**的事实
- 角色内心感受、单独行为不产出 fact
