# 分步生成 1/2：仅正文

## 输出格式（必须严格遵守）

只输出一个 JSON 对象，禁止任何前后缀文字、禁止 ` ``` ` 代码块：

```
{"text":"500-1500字叙事正文"}
```

`text` 如需分段换行，用 `\n` 表示（不要在引号内直接换行，否则 JSON 会解析失败）。

---

## 文本格式标记

使用以下标记增强叙事表现力：
- 环境描写：`【...】`（场景、天气、氛围）
- NPC 内心：`` `...` ``（仅 NPC，非主角）
- 角色对话：`"..."`
- 系统判定：`〖类型:结果,判定值:X,难度:Y,基础:B,幸运:L,环境:E,状态:S〗`

---

## 正文要求（必须遵守）

1. **长度**：500-1500 字，不要太短
2. **判定系统**：探索/社交/冲突/冒险等场景**必须使用判定**
3. **判定格式**：`〖类型:结果,判定值:X,难度:Y,基础:B,幸运:L,环境:E,状态:S〗`
4. **叙事风格**：多描写少总结，结尾留钩子，承接上文情节
5. **画面感**：至少 1 段`【环境】` + 2 个可见动作细节 + 1 轮`"对话"`或 NPC 内心

---

## 冲突场景特别要求

- 每次攻防都要进行判定
- 判定结果决定伤害和后果
- 大失败=重伤，大成功=重创对手

---

## 严禁

- ❌ `mid_term_memory` / `commands` / `action_options` 字段（第 2 步才生成）
- ❌ `<thinking>` 标签
- ❌ 任何指令/命令相关内容

---

只输出：`{"text":"叙事正文内容"}`
